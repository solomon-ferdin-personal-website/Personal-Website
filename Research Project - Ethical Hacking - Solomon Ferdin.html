Green Clothing Security Pentesting Assessment:

Solo-Sec Solutions performed a network assessment to test Green Clothing’s internal network security posture. Starting from an external perspective, the Solo-Sec team performed initial active reconnaissance research by dumpster diving through Green Clothing’s trash. To avoid any suspicion, we used an agent who dressed up as a homeless person to dig through the company trash in the back parking lot. There we found sensitive internal documents, including monthly employee work schedules, along with multiple employee emails and usernames. Next, we went online to Linkedin and Facebook to discover who were the company employees, with a goal to find out the admin-level managers who’s login credentials we could target and discover.


Scanning:

  • OS Fingerprinting and Vulnerability scans


Vulnerability scanners are used next to discover which OS operating system versions are used and which ports are open, to discover applications with weak passwords, and to send probes and requests to host to identify vulnerabilities. After performing a vulnerability scan, it was discovered that the Green Clothing network OS was a 15-year-old outdated Windows Vista version 2007. Only in Windows 10 and newer versions are security features built in to block many modern rootkits. Also, in the Payment Card Industry Standard PCI 11.2 requirement, it requires vulnerability scans 4 times a year, every quarter (Metrics). It was discovered that Green Clothing has not had a single vulnerability scan at present.


Gaining Access:

  • Performed a phishing attack on managerial employees from work schedules found in dumpster diving. Sent auto-deleting emails with malicious Excel documents showing new weekly updated work schedules. When employees click excel schedule document, they infect system with a 4-part malware package installed on temporary memory: A Point of Sale (POS) malware is installed, also a Security Software Disabling Trojan to disable network firewall and allow access to POS credit card transactions, a rootkit backdoor and keylogger are installed. (Council-EC, Types of Trojans). To cover tracks, after email is opened, the email auto-deletes.

  • A usb Bash Bunny is a usb drive that once inserted, it automatically and instantly releases penetration attacks and malware. Solo-Sec left multiple usb Bash Bunny’s laying around parking lot and inside store, just in case a curious employee picks up a usb drive and plugs it into a company computer. On the Bash Bunny was the 4 part malware package: POS malware, Security Software Disabling Trojan, Rootkit backdoor and keylogger. After one week, the usb’s are set to auto-delete all files on usb drive.

  • Also went to www.shodan.io to lookup any/all Green Clothing open ports that may be vulnerable for malicious access. Shodan showed the open ports of 9100, 515, 631. Using the PRET (Printer Exploitation Toolkit) found on Github (Rub-Nds), was able to hack into the printer’s open port 9100.


Gaining access to Green Clothing was not too difficult. First, we took the employee email addresses from the schedules found from dumpster diving, and we sent out an auto-deleting phishing email to those employees. The phishing email was disguised as a new employee work schedule. Originally Green Clothing’s server email address is greenclothing@gmail.com but Solo-Sec spoofed the company email address to greenclothing.schedules@gmail.com. From new spoofed email address, we attached an excel new work schedule document, which was exactly identical to current existing schedule, but instead the excel attachment schedule was a malicious link that auto-downloaded a malware package. The email also auto-deleted after execution.

The package contained 4 malicious downloads: (POS malware, Security Software Disabling Trojan, keylogger and a rootkit backdoor). The POS malware was used to gain access to all credit card transactions at the register. The Security Software Disabling Trojan was used to disable Green Clothing’s firewall from blocking malicious access traffic to the POS system. The keylogger was used to gain employee login credentials. And finally, the rootkit backdoor was used to maintain access to the network. The entire malware package was built to run in memory only, outside of the normal operating system, without anything actually being written to the disk file, so that it couldn’t be detected by the company’s anti-virus software located inside the operating system (Buster).

Solo-Sec also purposely left Bash Bunny usb drives lying around Green Clothing’s parking lot and inside the store as well. When inserted into any Green Clothing company computer, the usb also auto-downloaded the 4 part malware package. To clear track of any evidence, after one week, the usb’s also auto-deleted all data on usb drive.

To further gain access to Green Clothing’s network, Solo-Sec also hacked into their printers. We went to www.shodan.io to lookup any/all open ports for Green Clothing. Shodan showed the open ports of 9100, 515, 631. Found on Github, we used the PRET (Printer Exploitation Toolkit) to hack into the open port 9100 (Rub-Nds). With the PRET, we were able to gain access to the printer’s memory which saved a file of every document ever printed. After research into the memory of past documents printed, we were able to find documents containing network administrator usernames and passwords.

Access inside Green Clothing’s network was fully gained. Several admin-level employees reacted to the phishing email campaign and opened the malicious work-schedule links. Also, at least one Bash Bunny usb drive left in the parking lot or inside the store was inserted into a company computer. We wanted to test the employee login page to see if there was a login lockout enabled in case of multiple failed login attempts. It was discovered that there were unlimited login attempts allowed, which enabled attackers to perform brute force and dictionary attacks to crack passwords.

It was also discovered that all the POS credit card terminals were using the outdated unsecure swiping function. Swiping credit cards saves all card information locally in temporary memory. It’s recommended that the POS terminals are updated to use a chip-only function which uses a unique one-time code that is different for each and every credit card transaction, and thus cannot be captured and re-used.


Maintaining Access:

  • Continued access to Green Clothing’s network was allowed by the rootkit.


Solo-Sec was able to maintain network access through the rootkit backdoor. If Green Clothing re-power cycled off and on, or if they logged out of all accounts, the rootkit backdoor would still enable malicious network access. During the maintaining access phase, a low-profile is maintained and no further exploits are attempted. Solo-Sec continues to make a list of all POS credit card transactions and all Green Clothing internal sensitive data.


Clearing Tracks:

According to ITperfection in “Five Phases of Ethical Hacking” (Tech2020), an intelligent hacker clears all evidence by:

  • Clearing cache and cookies

  • Modifying registry values

  • Modifying/corrupting/deleting the values of Logs

  • Clearing out all Sent emails

  • Closing all open ports

  • Uninstalling all malware and applications used


According to the “Ethical Hacking” journal from the International Journal of Information Technology (Ushmani), “The last phase of hacking involves track clearing to erase all traces and evidence that the system was hacked. All the folders created, applications installed, and the registry values that were modified are deleted in this step ("Phases Of Hacking | Ethical Hacking"). The changes are made unrecognizable so that the hacking process is not detectable.”

In concluding the penetration test, the final step is to clear tracks and delete any/all evidence of network intrusion. The malware package (POS malware, Security Software Disabling Trojan, Rootkit and Keylogger) was deleted from the temporary memory. All the phishing emails auto-deleted after being opened. All the malware usb Bash Bunny’s auto-deleted their usb data after one week. The cache and cookies were deleted. All logs were deleted. All open ports were closed, including port 9100, 515, and 631. Finally, the security update best practice recommendations are provided to Green Clothing.


Penetration Test Security Update Best Practice Recommendations:



  • Shred all sensitive internal company documents and do not dump un-shredded documents in trash.

  • Train employees with basic entry-level cybersecurity best practices. Train to spot malicious phishing emails and to be very cautious of clicking any email links or downloads.

  • Train employees to never insert unauthorized media (usb sticks, hard-drives, external memory drives, etc..) into company computers.

  • Update Windows OS system to Windows 10 or newer version.

  • Perform 4 successful vulnerability scans per year (Per PCI 11.2 requirements).

  • Enable login lockout after 3-5 failed attempts.

  • Update POS terminal to use credit card chip instead of card swiping function.

  • Close all unnecessary unused open ports, including port 9100, 515, and 631.
